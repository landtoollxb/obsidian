# 文本处理与检索（grep/sed/awk）

## grep（查找）
```bash
grep "keyword" file
grep -n "keyword" file
grep -i "keyword" file
grep -RIn "keyword" /etc
grep -RIn --exclude-dir=.git --exclude="*.log" "keyword" .
```

### 常用组合
```bash
ps aux | grep java
ss -lntp | grep 9092
journalctl -u ssh -n 200 | grep -i fail
```

## less（分页查看）
```bash
less -R file
```
- 搜索：`/keyword`，下一个：`n`，上一个：`N`
- 退出：`q`

## awk（列处理/格式化）
```bash
awk '{print $1,$NF}' file
awk -F: '{print $1,$3}' /etc/passwd
df -h | awk 'NR==1 || $5+0 > 80 {print}'
```

## sed（替换/删除/插入）
```bash
sed 's/old/new/g' file
sudo sed -i 's/^#Port 22/Port 22/' /etc/ssh/sshd_config
sed -n '1,120p' file
sed -n '/BEGIN/,/END/p' file
```

## sort/uniq（排序去重）
```bash
sort file | uniq
sort file | uniq -c | sort -nr
```

## xargs（把输出变成参数）
```bash
cat files.txt | xargs -n 1 echo
find . -name "*.log" -print0 | xargs -0 -n 50 rm -f
```

## cut/tr（简单好用）
```bash
cut -d: -f1 /etc/passwd | head
echo "a,b,c" | tr ',' '\n'
```

## 实战片段

### 1) 从日志中提取错误并统计
```bash
grep -RIn "ERROR" /var/log | awk -F: '{print $1}' | sort | uniq -c | sort -nr | head
```

### 2) 批量替换配置项（先备份）
```bash
sudo cp app.conf app.conf.bak.$(date +%F_%H%M)
sudo sed -i 's/^port=.*/port=8080/' app.conf
```

## 常见坑
- `sed -i` 会直接修改文件，生产环境建议先备份。
- `grep -R` 在大目录可能很慢，优先缩小范围或排除目录。
