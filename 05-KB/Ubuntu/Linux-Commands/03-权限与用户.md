# 权限与用户

## 账号与身份
```bash
whoami
id
groups
```

## sudo 与切换用户
```bash
sudo -v
sudo -iu root
sudo -iu hadoop
```

## 用户与组管理（部署常用）
```bash
sudo useradd -m -s /bin/bash hadoop
sudo passwd hadoop
sudo groupadd bigdata
sudo usermod -aG bigdata hadoop
sudo usermod -aG sudo hadoop
```

## 文件权限模型（快速记忆）
- `r`=4，`w`=2，`x`=1
- `chmod 755 dir` = 所有人可读/可进目录，只有属主可写
- `chmod 644 file` = 所有人可读，只有属主可写

```bash
ls -l
chmod 755 /data
chmod 644 /etc/hosts
```

## 属主与属组
```bash
sudo chown user:group file
sudo chown -R hadoop:bigdata /data
```

## umask（新文件默认权限）
```bash
umask
```
- 常见：`022`（新文件 644，新目录 755）

## sudoers（谨慎）
```bash
sudo visudo
```

## 实战片段

### 1) 给大数据服务准备目录并授权
```bash
sudo mkdir -p /data/{hadoop,spark,kafka}/{data,logs}
sudo chown -R hadoop:bigdata /data
sudo chmod -R 755 /data
```

### 2) 快速查看某文件是谁的
```bash
stat /data/hadoop
```

## 常见坑
- 不要直接 `chmod -R 777` 解决权限问题，后续排查会更困难。
- 修改 `/etc/sudoers` 必须用 `visudo`，避免语法错误导致 sudo 不可用。
