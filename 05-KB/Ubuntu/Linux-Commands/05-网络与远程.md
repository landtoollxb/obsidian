# 网络与远程

## IP / 路由 / 网卡
```bash
ip a
ip r
ip link
```

## DNS / 解析（Ubuntu 24.04 常用）
```bash
resolvectl status
resolvectl query github.com
cat /etc/resolv.conf
```

## 端口与连接
```bash
ss -lntp
ss -lntup
ss -antp | head
sudo lsof -i :8080
```

## 连通性检查
```bash
ping -c 4 1.1.1.1
ping -c 4 github.com
curl -I https://github.com
curl -sS http://127.0.0.1:8080/health
```

## 下载与探测
```bash
wget -O file URL
curl -L -o file URL
nc -vz host 22
```

## SSH（部署必备）
```bash
ssh user@host
ssh -p 2222 user@host
ssh -i ~/.ssh/id_ed25519 user@host
```

### 免密登录（推荐走密钥）
```bash
ssh-keygen -t ed25519 -C "me@host"
ssh-copy-id user@host
```

### 常用 ssh 配置（本机）
文件：`~/.ssh/config`
```text
Host n1
  HostName 10.0.0.11
  User hadoop
  Port 22
  IdentityFile ~/.ssh/id_ed25519
```

## 复制与同步
```bash
scp file user@host:/data/
scp -r dir user@host:/data/
rsync -avP dir/ user@host:/data/dir/
```

## 防火墙/安全组联动提醒
- 端口通了但连不上：先看 `ss -lntp` 是否监听，再看 `ufw status`，最后看云厂商安全组。
