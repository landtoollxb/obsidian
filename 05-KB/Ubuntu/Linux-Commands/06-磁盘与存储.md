# 磁盘与存储

## 容量与占用
```bash
df -hT
du -sh /var/log/*
sudo du -xhd1 / | sort -h
```

## 磁盘识别
```bash
lsblk -f
blkid
sudo fdisk -l
```

## 挂载（临时）
```bash
sudo mkdir -p /data
sudo mount /dev/sdb1 /data
mount | grep /data
```

## 持久挂载（fstab）
1) 先拿 UUID：
```bash
lsblk -f
```
2) 编辑 `/etc/fstab`（示例，按你的 UUID/文件系统改）：
```text
UUID=xxxx-xxxx  /data  ext4  defaults,noatime  0  2
```
3) 校验（非常重要）：
```bash
sudo mount -a
df -hT | grep /data
```

## 文件系统扩容（常见：云盘扩容后）
视情况而定，常见 ext4：
```bash
sudo growpart /dev/sdb 1
sudo resize2fs /dev/sdb1
```

## 交换分区（swap）
查看：
```bash
swapon --show
free -h
```

## 大数据常见实践
- 数据盘独立挂载到 `/data`，避免把数据写满系统盘
- `noatime` 减少读写开销（按需）
- 目录权限：确保服务用户可读写
```bash
sudo chown -R hadoop:bigdata /data
sudo chmod -R 755 /data
```

## 常见坑
- 改 `/etc/fstab` 前先备份：`sudo cp /etc/fstab /etc/fstab.bak.$(date +%F_%H%M)`
- `mount -a` 必须通过，否则重启可能进维护模式。
