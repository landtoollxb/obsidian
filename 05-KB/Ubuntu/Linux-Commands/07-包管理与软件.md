# 包管理与软件（Ubuntu 24.04）

## apt（主力）
### 更新与安装
```bash
sudo apt update
sudo apt install -y curl wget git vim
```

### 升级
```bash
sudo apt upgrade -y
sudo apt full-upgrade -y
```

### 查包与信息
```bash
apt-cache search openjdk
apt show openssh-server
apt policy openssh-server
```

### 卸载与清理
```bash
sudo apt remove -y pkg
sudo apt purge -y pkg
sudo apt autoremove -y
sudo apt clean
```

## dpkg（离线/本地 deb）
```bash
sudo dpkg -i package.deb
sudo apt -f install
```

## snap（Ubuntu 常见）
```bash
snap list
sudo snap install yq
sudo snap remove yq
```

## 常用软件建议（部署/排障）
```bash
sudo apt install -y \
  openssh-server \
  curl wget ca-certificates \
  net-tools iproute2 \
  dnsutils \
  rsync \
  unzip zip \
  jq \
  lsof \
  htop \
  sysstat
```

## Java（大数据部署常用）
Ubuntu 24.04 常见安装：
```bash
sudo apt update
sudo apt install -y openjdk-17-jdk
java -version
```

设置 JAVA_HOME（示例）：
```bash
readlink -f $(which java)
```
然后把 JAVA_HOME 写入 `/etc/profile.d/java.sh`（按实际路径改）：
```bash
echo 'export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64' | sudo tee /etc/profile.d/java.sh
echo 'export PATH=$JAVA_HOME/bin:$PATH' | sudo tee -a /etc/profile.d/java.sh
source /etc/profile.d/java.sh
```

## 常见坑
- `dpkg -i` 失败通常是依赖缺失，用 `sudo apt -f install` 修复。
- `snap` 与 `apt` 的软件来源不同，遇到路径/权限问题先确认安装方式。
