# 安全与防火墙

## ufw（最常用）
```bash
sudo ufw status verbose
sudo ufw enable
sudo ufw disable
```

### 放行/拒绝端口
```bash
sudo ufw allow 22/tcp
sudo ufw allow 8080/tcp
sudo ufw deny 9092/tcp
sudo ufw delete allow 8080/tcp
```

### 按网段放行（内网集群常用）
```bash
sudo ufw allow from 10.0.0.0/24 to any port 22 proto tcp
```

## SSH 安全（建议）
### 查看 sshd 配置与状态
```bash
sudo systemctl status ssh -l
sudo sshd -T | head
```

### 常见加固点（按需）
- 禁止 root 远程（改前确保你有 sudo 用户）：`PermitRootLogin no`
- 禁止密码登录（改前确保密钥可用）：`PasswordAuthentication no`

改完后：
```bash
sudo systemctl restart ssh
```

## 用户与最小权限
- 普通服务用独立用户运行（不要用 root）
- 目录权限精确到服务用户（配合 `chown/chmod`）

## fail2ban（可选）
如果暴露公网 SSH，建议安装：
```bash
sudo apt update
sudo apt install -y fail2ban
sudo systemctl enable --now fail2ban
sudo fail2ban-client status
```

## 常见坑
- 防火墙放行了但还是连不上：检查云安全组/路由/NAT，再用 `ss -lntp` 确认服务确实在监听。
- 改 sshd 配置前先开一个“备用终端会话”，避免把自己锁在门外。
