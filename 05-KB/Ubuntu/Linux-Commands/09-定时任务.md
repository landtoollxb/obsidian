# 定时任务（cron / systemd timers）

## cron（简单直观）
### 编辑当前用户的 crontab
```bash
crontab -e
crontab -l
```

### 示例
- 每 5 分钟执行一次：
```text
*/5 * * * * /usr/local/bin/backup.sh >> /var/log/backup.log 2>&1
```
- 每天 03:30 执行：
```text
30 3 * * * /usr/local/bin/rotate.sh
```

### 常见位置
- 系统级：`/etc/crontab`
- 目录：`/etc/cron.d/`、`/etc/cron.daily/` 等

## systemd timers（更适合服务化/可追踪）
### 1) 创建 service
`/etc/systemd/system/myjob.service`
```ini
[Unit]
Description=My periodic job

[Service]
Type=oneshot
ExecStart=/usr/local/bin/myjob.sh
```

### 2) 创建 timer
`/etc/systemd/system/myjob.timer`
```ini
[Unit]
Description=Run myjob periodically

[Timer]
OnBootSec=2min
OnUnitActiveSec=10min
Persistent=true

[Install]
WantedBy=timers.target
```

### 3) 启用
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now myjob.timer
systemctl list-timers --all | grep myjob
journalctl -u myjob.service -e
```

## 常见坑
- cron 默认环境变量少，脚本里最好写绝对路径。
- systemd timer 更容易追踪日志与执行结果，适合部署任务。
