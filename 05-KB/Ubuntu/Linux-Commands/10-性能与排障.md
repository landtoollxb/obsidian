# 性能与排障

## 一条“通用排障链路”
1) 服务是否在跑：`systemctl status xxx -l`  
2) 日志报什么：`journalctl -u xxx -e`  
3) 端口是否监听：`ss -lntp | grep :PORT`  
4) 机器资源是否吃满：`top` / `free -h` / `df -h`  
5) 网络/DNS 是否正常：`ip a` / `ip r` / `resolvectl status` / `curl -I`  

## 系统概览
```bash
uname -a
lsb_release -a
uptime
free -h
df -hT
```

## CPU / 内存 / 负载
```bash
top
ps -eo pid,user,%cpu,%mem,cmd --sort=-%cpu | head
ps -eo pid,user,%cpu,%mem,cmd --sort=-%mem | head
vmstat 1 5
```

## 磁盘 IO（需要 sysstat）
```bash
sudo apt update
sudo apt install -y sysstat
iostat -xz 1 5
```

## 网络排障
```bash
ip a
ip r
ss -lntp
ss -antp | head
ping -c 4 1.1.1.1
curl -I https://github.com
```

## 端口与进程定位
```bash
sudo lsof -i :8080
sudo lsof -i -P -n | head
```

## 文件句柄/连接数（大数据常见瓶颈）
```bash
ulimit -n
cat /proc/sys/fs/file-max
sudo lsof -p PID | wc -l
```

## 日志与 systemd
```bash
systemctl status xxx -l
journalctl -u xxx -n 200
journalctl -u xxx -f
journalctl -p err -b
```

## strace（深入排障，慎用）
```bash
sudo apt install -y strace
sudo strace -p PID -f -s 128 -o /tmp/trace.log
```

## 常见“报错 → 快速定位”
- “端口被占用”：`ss -lntp | grep :PORT` 或 `lsof -i :PORT`
- “磁盘满”：`df -hT` + `du -xhd1 / | sort -h`
- “权限不足”：`ls -l` + `stat` + `id` + `chown/chmod`
- “DNS 不通”：`resolvectl status` + `resolvectl query 域名`
