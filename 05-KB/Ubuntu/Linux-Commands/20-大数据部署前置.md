# 大数据部署前置（Ubuntu 24.04）

目标：把“部署前必做的系统准备”固化为可复制的命令清单，减少反复查命令。

## 1) 主机名 / hosts / DNS
```bash
hostnamectl
sudo hostnamectl set-hostname node1
cat /etc/hosts
```

建议：集群内用固定内网 IP + hosts（至少保证关键节点能互相解析）。

DNS 排查：
```bash
resolvectl status
resolvectl query node1
```

## 2) 时间同步（chrony）
```bash
timedatectl
timedatectl status
```

安装与启用：
```bash
sudo apt update
sudo apt install -y chrony
sudo systemctl enable --now chrony
chronyc sources -v
chronyc tracking
```

## 3) SSH 免密（部署/运维必备）
在控制机生成密钥并分发：
```bash
ssh-keygen -t ed25519 -C "bigdata"
ssh-copy-id hadoop@node1
ssh-copy-id hadoop@node2
```

验证：
```bash
ssh hadoop@node1 "hostname; date"
```

## 4) 端口检查（部署前先确认）
```bash
ss -lntp
sudo lsof -i :8080
nc -vz node1 22
```

## 5) 防火墙（ufw）与放行策略
```bash
sudo ufw status verbose
sudo ufw allow 22/tcp
```

集群内网放行示例：
```bash
sudo ufw allow from 10.0.0.0/24
```

## 6) swap（按平台要求决定）
查看：
```bash
swapon --show
free -h
```

临时关闭：
```bash
sudo swapoff -a
```

永久关闭：编辑 `/etc/fstab` 注释 swap 行（谨慎，改前备份）。

## 7) 文件句柄 / 进程数 / limits（非常常见）
查看当前 shell：
```bash
ulimit -n
ulimit -u
```

建议为服务用户配置（示例：`/etc/security/limits.d/99-bigdata.conf`）：
```text
hadoop soft nofile 65535
hadoop hard nofile 65535
hadoop soft nproc  65535
hadoop hard nproc  65535
```

确认新会话生效：
```bash
sudo -iu hadoop
ulimit -n
```

## 8) sysctl（内核参数，按需）
常用关注点：文件句柄、网络队列、端口范围。

查看：
```bash
sysctl fs.file-max
sysctl net.ipv4.ip_local_port_range
```

写入（示例：`/etc/sysctl.d/99-bigdata.conf`）：
```text
fs.file-max = 2097152
net.ipv4.ip_local_port_range = 10240 65535
net.core.somaxconn = 1024
```

加载：
```bash
sudo sysctl --system
```

## 9) 磁盘挂载与目录规范（强烈建议）
目标：数据盘挂到 `/data`，并把各组件的数据/日志放到其下，避免系统盘被写满。

```bash
lsblk -f
sudo mkdir -p /data
sudo mount /dev/sdb1 /data
df -hT | grep /data
```

目录与权限（示例）：
```bash
sudo groupadd -f bigdata
sudo useradd -m -s /bin/bash -g bigdata hadoop || true
sudo mkdir -p /data/{hadoop,spark,kafka}/{data,logs}
sudo chown -R hadoop:bigdata /data
sudo chmod -R 755 /data
```

## 10) Java（多数大数据组件依赖）
```bash
sudo apt update
sudo apt install -y openjdk-17-jdk
java -version
```

JAVA_HOME（示例）：
```bash
echo 'export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64' | sudo tee /etc/profile.d/java.sh
echo 'export PATH=$JAVA_HOME/bin:$PATH' | sudo tee -a /etc/profile.d/java.sh
```

## 11) systemd 服务模板（把组件当服务管理）
参考：[[04-进程与服务-systemd]]

最小流程：
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now your-service
systemctl status your-service -l
journalctl -u your-service -f
```

## 12) 快速验收清单（复制执行）
```bash
hostnamectl
timedatectl
chronyc tracking || true
ss -lntp | head
df -hT
ulimit -n
java -version || true
```
